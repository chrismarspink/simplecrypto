Verify Certificate

Certificate issuance process:
  1. Converts data (Data) containing serial number, issuer, public key, etc. into hash value.
  2. An authorized authority signs the hash value with its own private key saying "I will prove this data".
  3. Then, a certificate is issued by combining the signed value (Signature) + data (Data).
In the certificate verification stage
  1. The web browser receives the certificate passed from the server.
  2. The certificate has the public key of the higher authority, and the hash value is obtained by decrypting the signature.
  3. Hash value is obtained through the hash function of data existing in the certificate.
  4. Verification of the certificate is performed by comparing the hash values of the two.
